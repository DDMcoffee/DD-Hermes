# DD Hermes 指挥文档

这个目录只放给当前主线程和项目负责人阅读的中文文档。

第一次看当前 DD Hermes 体验版状态时，先运行 `./scripts/demo-entry.sh`，再读这里的 `06` 和 `07`。

## 文件

- `01-项目目标对齐.md`
  - 解释 DD Hermes 一期到底在做什么、不做什么，以及当前项目级目标。
- `02-三层终点定义.md`
  - 把 execution slice、task、phase 三层终点写成统一规范。
- `03-执行线程干到底说明.md`
  - 给代码执行线程的中文收尾文档，说明它下一轮该做到哪里才停。
- `04-任务重校准与线程策略.md`
  - 记录当前主目标、任务拆分，以及什么时候需要或不需要第二线程。
- `05-体验版本路线图.md`
  - 说明 DD Hermes 一期最小可体验闭环、当前 blocker，以及什么时候能看到体验版本。
- `06-一期PhaseDone审计.md`
  - 说明一期已经完成到哪里、为什么还没 `phase done`、以及当前剩余 blocker。
- `07-体验入口任务说明.md`
  - 记录 `dd-hermes-demo-entry-v1` 这条已完成主线任务的历史说明。
- `08-恒定锚点策略.md`
  - 定义 phase-2 当前主线：把产品锚点与质量锚点变成协议层常驻席位。
- `09-今日输入整理.md`
  - 只整理你今天明确输入过的要求、偏好和判断，不混入我方实现结论。

## 使用方式

- 架构或方向判断先看 `01-项目目标对齐.md`。
- 讨论“什么时候算完成”先看 `02-三层终点定义.md`。
- 要进入实现阶段时，直接把 `03-执行线程干到底说明.md` 作为当前线程内的实现基线。
- 要判断“现在到底该做什么、要不要额外开新聊天线程”时，先看 `04-任务重校准与线程策略.md`。
- 要判断“什么时候算出现体验版本、当前还差什么”时，先看 `05-体验版本路线图.md`。
- 要判断“一期现在到底是不是已经完成”时，先看 `06-一期PhaseDone审计.md`。
- 要看当前阶段路线时，先看 `05-体验版本路线图.md`；`07-体验入口任务说明.md` 只作为已完成任务的历史说明保留。
- 要看 phase-2 当前主线怎么推进，直接看 `08-恒定锚点策略.md`。
- 要回看你今天到底明确说过哪些要求，直接看 `09-今日输入整理.md`。
