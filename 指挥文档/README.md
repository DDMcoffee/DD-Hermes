# DD Hermes 指挥文档

这个目录只保留给当前主线程和项目负责人阅读的中文入口文档。

硬规则：

- 这里的 Markdown 文件数长期不得超过 `7`。
- 新判断优先并入现有页面，不再为同一主题额外开新页。

第一次看当前 DD Hermes 状态时，先运行 `./scripts/demo-entry.sh`，再读 `06` 和 `04`。

## 当前文件

- `01-项目目标对齐.md`
  - 解释 DD Hermes 为什么存在、本期做什么、不做什么。
- `02-三层终点定义.md`
  - 统一 execution slice、task、phase 三层终点，并收口实现阶段的进入条件与停止条件。
- `04-任务重校准与线程策略.md`
  - 记录当前唯一主线、任务拆分、是否需要第二线程，以及当前线程如何使用多 agent。
- `06-一期PhaseDone审计.md`
  - 回答“现在能不能用 DD Hermes、能用到什么程度、一期为什么已经完成、下一步是什么”。
- `08-恒定锚点策略.md`
  - 定义 `Product Anchor` / `Quality Anchor` 的协议地位，并回答它们是否等于长期在线子 agent。
- `09-今日输入整理.md`
  - 只整理你明确输入过的要求、偏好和判断，不混入我方实现结论。

## 阅读顺序

1. 先运行 `./scripts/demo-entry.sh`
2. 再读 `06-一期PhaseDone审计.md`
3. 要判断当前线程和任务怎么推进，再读 `04-任务重校准与线程策略.md`
4. 要判断“什么时候算完成”，再读 `02-三层终点定义.md`
5. 要看 phase-2 主线与锚点语义，再读 `08-恒定锚点策略.md`
6. 要回看长期北极星，再读 `01-项目目标对齐.md`
7. 要回看你今天明确说过哪些要求，再读 `09-今日输入整理.md`
