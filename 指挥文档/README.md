# DD Hermes 指挥文档

这个目录只保留给当前主线程和项目负责人阅读的中文入口文档。

硬规则：

- 这里的 Markdown 文件数长期不得超过 `7`。
- 新判断优先并入现有页面，不再为同一主题额外开新页。

## 当前文件

- `01-项目目标对齐.md`
  - 项目级北极星和文档分层，不讲当前任务细节。
- `02-三层终点定义.md`
  - execution slice、task、phase 三层完成定义。
- `04-任务重校准与线程策略.md`
  - 当前唯一主线、当前下一步、线程裁决和继续开发起点。
- `06-一期PhaseDone审计.md`
  - 回答“现在能不能用 DD Hermes、一期完成到了哪里、phase-2 当前处于什么位置”。
- `08-恒定锚点策略.md`
  - 只定义 `Product Anchor` / `Quality Anchor` 的长期协议地位。
- `09-今日输入整理.md`
  - 历史附录；记录你明确输入过的要求、偏好和判断，不是当前主线状态页。

## 阅读顺序

1. 先运行 `./scripts/demo-entry.sh`
2. 再读 `06-一期PhaseDone审计.md`
3. 要判断当前线程和任务怎么推进，再读 `04-任务重校准与线程策略.md`
4. 要判断“什么时候算完成”，再读 `02-三层终点定义.md`
5. 要看 phase-2 主线与锚点语义，再读 `08-恒定锚点策略.md`
6. 要回看长期北极星，再读 `01-项目目标对齐.md`
7. `09-今日输入整理.md` 只在需要追溯历史输入时再看。
