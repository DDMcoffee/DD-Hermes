# DD Hermes 体验入口任务说明

## 任务标识

- `task_id`: `dd-hermes-demo-entry-v1`
- 任务性质：下一条主线实现任务
- 推荐执行形态：`Lead + Executor + Skeptic`

说明：

- 这不是新的 runtime 任务。
- 这不是新的 scheduler 任务。
- 这不是继续扩写控制面。
- 这是把已经存在的体验版 v0 做成一个用户可见入口。

## 任务目标

让一个不想读大量内部文档的人，也能快速看到：

1. DD Hermes 现在已经做到了什么。
2. 最近一次真实 end-to-end 证明是什么。
3. 当前还没做到什么。
4. 下一条主线任务是什么。

## 建议交付物

### 1. 一个只读入口脚本

建议路径：

- `scripts/demo-entry.sh`

这个脚本至少要输出：

- 当前体验版状态
- 最近一次真实证明的 `task_id`
- 对应的 execution / integration 证据锚点
- 当前主线任务
- 指向中文入口页的路径

### 2. 一页中文入口说明

建议路径：

- 可以新增一页中文 Markdown
- 也可以把现有中文指挥文档中的某一页升级成入口页

但必须满足：

- 单页可读
- 不是工程师内部 trace dump
- 能给用户一个“这就是当前体验版”的直观入口

## 完成标准

任务完成时，至少要满足：

1. 一个命令足以看到体验版当前状态。
2. 一页中文文档足以解释这个状态。
3. 输出引用真实仓库事实，而不是硬编码宣传语。
4. 这条任务本身能被正常收成 `done/archive`。

## 非目标

- 不做新的 provider / gateway / runtime。
- 不做自动续跑系统。
- 不把入口脚本变成新的调度器。
- 不把这个任务扩成“重新设计 DD Hermes”。

## 为什么这条任务是下一条主线

因为现在真正缺的不是“能力”，而是“可见性”。

仓库已经证明过一次真实体验版闭环，但用户还不能用一个简单入口看到它。`dd-hermes-demo-entry-v1` 是最小、最真实、最不跑偏的下一步。
