# DD Hermes 产品介绍与使用说明

这页回答两个问题：

- DD Hermes 是什么
- 第一次接手或继续推进时，最短应该怎么用

## 1. 一句话产品介绍

DD Hermes 不是新的 agent runtime，也不是新的聊天产品。

它是一个面向 Codex IDE 的工程 harness / 控制面，用显式的：

- `contract`
- `state`
- `context`
- `handoff`
- `worktree`
- `verification`

把复杂 agent 工作从“靠聊天协调”变成“可规划、可反证、可执行、可验收”的工程流程。

## 2. 它解决什么问题

如果没有 DD Hermes，复杂任务很容易出现这些问题：

- 任务边界只存在聊天里，过几轮就漂
- 执行线程写了代码，但没人知道是不是已经验收
- memory、state、runtime、handoff 混在一起
- 一个执行结果留在 worktree 里，没有真正集成和归档
- 现在到底有没有 active mainline，要靠人手工 sweep 仓库

DD Hermes 的作用，就是把这些判断压进仓库内的协议和工件里。

## 3. 它不是什么

明确不是：

- 新的 Hermes runtime
- 新的 provider / gateway / plugin loader
- 一个强调“聊得像产品”而不是“做得像工程”的 agent 壳
- 用聊天总结替代 memory 系统的文档工程

## 4. 适合怎么用

最适合三类场景：

1. 复杂任务要先规划、再拆分、再执行
2. 一个任务需要留下明确的验收与归档证据
3. 多角色/多 worktree 内部协作，但对外仍保持一个主线程入口

如果只是一次性小修改，DD Hermes 也能用，但价值主要体现在复杂工程任务上。

## 5. 最短使用路径

### 情况 A：第一次了解这个项目

1. 读本页
2. 读 `01-项目目标对齐.md`
3. 运行 `./scripts/demo-entry.sh`
4. 读 `06-一期PhaseDone审计.md`
5. 再读 `04-任务重校准与线程策略.md`

### 情况 B：继续推进开发

1. 先运行 `./scripts/demo-entry.sh`
2. 看当前有没有 active mainline
3. 如果有，就读对应 `workspace/contracts/<task_id>.md`
4. 再看 `workspace/state/<task_id>/state.json`
5. 进入执行前，再看 handoff / closeout / verification

### 情况 C：当前没有 active mainline

1. 不要直接拍脑袋立新主线
2. 先读 `04-任务重校准与线程策略.md`
3. 看最近 proof 的 archive
4. 只在 repo evidence 明确支持时，才启动下一条窄主线

## 6. 使用时的几个硬判断

### 先哲学，中规划，后执行

先确认这条任务到底服务什么产品目标，再决定如何拆分、谁执行、怎么验收。

### 对抗式思辨不是装饰

复杂问题默认要先留反证位、做多视角讨论，而不是直接把第一反应写成代码。

### 一个主线程，对外尽量简单

用户默认只看一个指挥线程。
内部可以有多角色、多 worktree、多证据链，但不要把复杂度泄漏到外部入口。

### 完成不是“代码写完”

真正完成至少要同时看：

- 代码
- handoff
- state
- verification
- git integration

## 7. 继续开发时最常用的入口

- `./scripts/demo-entry.sh`
  - 看当前状态、最近 proof、是否存在 active mainline
- `指挥文档/04-任务重校准与线程策略.md`
  - 看当前下一步到底该做什么
- `指挥文档/06-一期PhaseDone审计.md`
  - 看“现在能不能用、当前做到哪”
- `workspace/contracts/`
  - 看任务边界
- `workspace/state/`
  - 看最新控制面真相
- `workspace/handoffs/` / `workspace/closeouts/`
  - 看执行切片到底做到哪

## 8. 一个实际判断标准

如果你在使用 DD Hermes 时，已经不需要靠聊天历史去回答下面这些问题，说明它在起作用：

- 这个任务的边界是什么
- 现在谁在负责
- 到底有没有 active mainline
- 这条执行切片有没有真正完成
- 最近一次 proof 到底证明了什么

如果这些问题还必须靠聊天回忆来答，说明控制面还没有收口好。
