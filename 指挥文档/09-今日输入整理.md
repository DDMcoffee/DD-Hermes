## 3. 工作方式要求

- 复杂问题必须优先使用多 agent 讨论，不要单线程直接拍板。
- 当前线程被定义为“指挥线程”。
- 代码执行可以放到独立线程，但是否启用要由指挥线程裁决。
- 你强调：
  - 每次进入复杂分析或重大判断时，要明确说明“正在用多 agent”。
  - 指挥线程应该负责规划、裁决、收口，而不是随意漂移到执行面。

## 4. 线程与 Agent 要求

- 你想过的理想形态包括：
  - 一个指挥线程，专注讨论、框架、目标和验收。
  - 一个代码执行线程，专注 coding。
  - agent 管理 agent，agent 使用 Codex。
  - 可以设定持续运行目标，例如跑 5 小时，额度触发时暂停，恢复后继续。
- 但你后面也明确担心：
  - 多线程容易跑偏。
  - 如果另一个线程已经偏离主线，指挥线程必须收回来裁决。
