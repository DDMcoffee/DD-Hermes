## Progress Note

- 今天的 DD Hermes 像是在把“我觉得可以了”改成“我把判决写进账本里”。以前很多门卫判断靠临时推导；今天开始，关键 verdict 要落在 `state.json` 里，谁来接手都能直接读到“卡在哪、为什么卡、什么时候更新过”。
- 我们还学会了一件更像“班规”的事：质量位不再只靠口头说“有人看过”，而是要么独立（independent），要么明确降级（degraded）并写清理由；另外，任务难度（`T0~T4`）也要决定是不是必须升级到独立质量位。
- 到了晚上，仓库还做了一个“诚实整理”：宁可说“现在没有 active mainline”，也不要让过期候选（被吸收的提案、暂停实验）继续假装还在排队等执行。

## Confirmed Patterns

- “判决要可恢复”：把 gate 真相从散落的 bool/推导，压到 `state.verdicts`，并让 `state.read / context.build / dispatch / quality.gate` 复用同一层 truth。
- “质量位要可解释”：`independent` vs `degraded` 不是心情，是控制面字段 + 原因 + 更新时间。
- “难任务先升级”：用 `T0~T4` 把升级规则写死，避免高回归任务在 degraded 状态下偷偷推进。
- “宁可空缺也别伪造”：archive 可以选择“不立 successor”；仓库入口要能诚实输出 `no active mainline`。
- “完成不是口号”：`execution_closeout` 进入 verdict 层后，才能把“模板填过”与“证据可验收”拆开。
